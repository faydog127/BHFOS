# Codex audit template

Objective:
Audit this system and tell me its actual state.

Scope:
[Files, app, workflow, or deployment area.]

Audit questions:
- What is working?
- What is unproven?
- What are the top failure points?
- What is the current maturity stage?
- What is the next hardening step?

Instructions:
- inspect before concluding
- separate facts from assumptions
- do not mark anything production-ready without evidence
- prefer concrete failure modes over generic warnings

Output format:
1. current state
2. what is proven
3. what is not proven
4. risks
5. recommended next step
