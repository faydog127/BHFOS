# 05_assumption_verification_report.md

| Assumption | Status | Evidence |
|------------|--------|----------|
| Line item tables `quote_items` and `invoice_items` exist and public quote/invoice functions return them | **Confirmed** | `public-quote` selects `quote_items` and returns them in payload (`supabase/functions/public-quote/index.ts:64-109`); `public-invoice` selects `invoice_items` in its response (`supabase/functions/public-invoice/index.ts:61-101`); CRM builders persist the same tables (`src/pages/crm/QuoteBuilder.jsx:53-158`, `InvoiceBuilder.jsx:71-233`). |
| `public-quote` logs to `public.public_events` and uses `public_token` | **Confirmed** | `public-quote` requires `public_token`, filters quotes by it, and calls `logPublicEvent` on both success and failure (`supabase/functions/public-quote/index.ts:29-118`); `logPublicEvent` writes hashed tokens to `public.public_events` (`supabase/functions/_shared/publicUtils.ts:6-62`). |
| `public-invoice` logs to `public.public_events` and uses `public_token` | **Confirmed** | `public-invoice` bases lookups on `public_token` and logs `public_invoice_view` events via `logPublicEvent` (`supabase/functions/public-invoice/index.ts:34-103`). |
| `public-quote-approve` updates acceptance/decline timestamps and closes follow-up tasks | **Confirmed** | After patching `quotes` with `accepted_at`/`rejected_at`, the function updates `crm_tasks` rows matching follow-ups (`supabase/functions/public-quote-approve/index.ts:67-163`). |
| `public-pay` invokes RPC `process_public_payment`, updates invoice paid fields, inserts into `transactions`, and logs `public_events` | **Confirmed** | Calls `supabaseAdmin.rpc('process_public_payment', ...)`, updates `invoices` (`paid_at`, `amount_paid`, `balance_due`), inserts `transactions` record, and logs `public_pay` events (`supabase/functions/public-pay/index.ts:86-188`). |
| There is a `crm_tasks` table with source/priority metadata | **Confirmed** | Migration `supabase/migrations/20260110_add_followup_tasks.sql:1-23` adds tenant/source/type/priority notes columns plus indexes needed for follow-up task lookup. |
| No authoritative internal `events` table exists yet (completeness vs Appendix A.2) | **Confirmed** | Only `public.public_events` is defined; `rg -n "CREATE TABLE public\\.events" supabase/migrations` returns no matches, so the broader event taxonomy in Appendix A.4 is not captured yet. |
| Job statuses currently omit `UNSCHEDULED` (or UI hides “jobs to schedule”) | **Confirmed** | Jobs UI/logic only references `scheduled`, `in_progress`, `completed`, `on_hold`, `pending_schedule`, etc. (`src/pages/crm/Jobs.jsx:43-130`), and `rg -n \"UNSCHEDULED\" -S C:\\BHFOS\\command-center` returns no occurrences. |
| Business hours config exists but is not enforced across outbound automation | **Partially Confirmed** | `global_config` table exists (`supabase/migrations/20251210_create_core_tables.sql:26-52`) but contains only `system_status`/`maintenance_mode`; no `business_hours` key is seeded and `rg -n \"business_hours\" -S supabase src` returns nothing, so the automation flows still hard-code `getNextBusinessDayAtTen` without reading tenant config. |
