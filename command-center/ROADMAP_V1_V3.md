## V2 — Operational Integrity + Field Execution

### V2 target outcomes

* Every job moves cleanly from **lead → quote → job → appointment/work**
* Technicians can perform inspections with **clear structure and no guessing**
* Statuses are trustworthy
* Owners of each mutation path are enforced
* UI and backend tell the same truth

### V2 major workstreams

#### 1. Lifecycle clarity

* eliminate confusion across lead / quote / job / appointment flow
* tighten owner paths
* reduce raw writes
* lock transition logic further

#### 2. Inspection execution

* define the inspection entity boundary correctly
* lock vocabulary before building
* create a technician-friendly inspection workflow
* ensure reports and outputs are consistent

#### 3. Tenant/security integrity

* H1b tenant immutability
* continue service-role audit
* keep trust boundaries tight

#### 4. UI truth alignment

* remove remaining places where UI can drift from canonical status
* make workflows operationally obvious

### V2 success definition

By the end of V2:

* office/admin side knows exactly where every opportunity is
* technicians know exactly what to do in the field
* no one is guessing what state a record is in
* inspections are executable, not improvised
